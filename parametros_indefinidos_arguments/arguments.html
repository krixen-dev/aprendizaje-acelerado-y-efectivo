Parámetros indefinídos

En JavaScript, es posible definir funciones que acepten un número variable de parámetros, incluso cuando no se especifica la cantidad exacta de argumentos que se pasarán. Esto se logra utilizando el objeto especial arguments o mediante el uso del operador de propagación (...).

function suma() {
let resultado = 0;

for (let i = 0; i < arguments.length; i++) {
resultado += arguments[i];
}

return resultado;
}

console.log(suma(2, 3, 5, 8)); // Output: 18
En este ejemplo, la función suma acepta cualquier cantidad de argumentos y los suma utilizando el objeto arguments.

Utilizando el operador de propagación (...):

A partir de ECMAScript 6 (ES6), puedes utilizar el operador de propagación (...) para recoger los argumentos en un array.

function suma(...numeros) {
return numeros.reduce((total, num) => total + num, 0);
}
console.log(suma(2, 3, 5, 8)); // Output: 18
En este caso, la función suma acepta cualquier cantidad de argumentos y los recoge en el array numeros utilizando el operador de propagación. Ambos enfoques te permiten trabajar con un número variable de parámetros. Sin embargo, el uso del operador de propagación es más moderno y generalmente preferido debido a su sintaxis más clara y a que arguments tiene algunas limitaciones y no es un array real. Es importante destacar que, si bien estas técnicas permiten trabajar con un número variable de parámetros, es recomendable documentar claramente cómo deben usarse tus funciones para que los usuarios comprendan qué argumentos son opcionales o necesarios.
